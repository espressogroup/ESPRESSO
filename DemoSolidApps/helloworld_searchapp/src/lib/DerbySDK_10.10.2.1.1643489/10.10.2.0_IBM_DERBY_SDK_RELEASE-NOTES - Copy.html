Overview

IBM Derby SDK 10.10.2.0.1582446 is based on the Apache Derby release 10.10.2.0.

The Derby SDK release differs from the Apache Derby release in the following ways:

  • The packaging is different, which shows, among other things, in the following ways:
    • The IBM Derby SDK does not include source code.
    • IBM Derby SDK does not include debug code, although this can be provided on demand for analysis of problem situations.
    • The IBM Derby SDK supports the servlet code only as a demo. derby.war has been moved to the 'servlet' directory in the demo directory. However, the servlet documentation is still part of the full documentation.
    • The IBM Derby SDK is only provided as zip files (the Apache product comes in tgz files as well).
    • The IBM Derby SDK is provided in two parts; a _core.zip, holding the jar files and release notes, and a _extra.zip, holding all other files.
    • The IBM Derby SDK does not automatically include jars for running the tests. These can be provided separately if desired.
  • The previous Derby SDK release was based on Apache Derby release 10.8. There were fixes and features introduced between Apache Derby 10.8 and Apache Derby 10.10.2.0 which were released as Apache Derby 10.9 and Apache Derby 10.10.1.1. Only a select number of the bug fixes were backported to later pointreleases of IBM Derby SDK 10.8. For an overview of all fixes, refer to the Release Notes in the releasenotes section:
    • 10.9.1.0_RELEASE-NOTES.html
    • 10.10.1.1_RELEASE-NOTES.html
    • 10.10.2.0_RELEASE-NOTES.html

Apache Derby is a pure Java relational database engine using standard SQL and JDBC as its APIs. More information about Derby can be found on the Apache web site. Derby functionality includes:

  • Embedded engine with JDBC drivers
  • Network Server
  • Network client JDBC drivers
  • Command line tools: ij (SQL scripting), dblook (schema dump) and sysinfo (system info)

Java and JDBC versions supported:

  • Java SE 6 and higher with JDBC 4.1 and 4.2.
  • Java ME CDC/Foundation Profile 1.1 with JSR-169 JDBC Optional Package for CDC/Foundation Profile.

Support Issues

The team supporting the IBM Derby SDK common component does not support all features of Apache Derby SDK. Specifically, the following is not supported:

  • Builtin Authentication. Even though there have been changes to Embedded built-in authentication, this is not secure. You should use LDAP or a custom authentication scheme.
  • Locale files. Although the translated message files are included in the distributions, these were not created through the IBM localization services and there is no guarantee attached.
  • Accessibility. Although the basic software only outputs standard text or creates only standard text, neither the javadoc, nor the docs meet IBMs accessibility complience rules. If an embedding product desires to both include docs as well as the core functionality, they either must document a deviation from accessibility, or create their own dedicated documentation.
  • Replication. The IBM Derby SDK team does not support Apache Derby Replication. This feature is Incomplete. Use other IBM Replication solutions.
  • Java 5. The IBM Derby SDK team does not support Java 5 with this version of Derby.
  • SQL ROW_NUMBER(). Incomplete feature. Performs poorly.
